Forms

Forms are used to collect user input. To create forms in HTML, the <form></form> tag is used. There are 3 attributes used in the form tag. They are the action, method and enctype attribute.

The action attribute tells the browser where to take the user after the user submit the form. This attribute also tells the browser where to process the data of a form. The value for the action attribute is a path of a destination.It can be an URL of just a path. You will learn how the data is processed in PHP.

The method attribute specifies whether of not the data should be present in the URL after the user submit his/her data. There are only 2 values for the method attribute--post and get.
When method="get", all the data will show in the URL and all of them will be processed on the web page the action attribute specifies.
When method="post", all the data will be sent to the server to be processed and no data will be shown in the URL. This is very useful when you are going to process sensitive data for example passwords, e-mail, handphone number etc.

The enctype attribute tells the server how the data of the form should be encoded. Another way of saying is what type of data is present in the form and how should they be processed. The most commom value is "multipart/form-data" and this will let the broswer know that there is a file upload. Using enctype="multipart/form-data" in a form which has no file uploads will not cause any errors.

User Inputs

To create a user input(text box), use the <input> tag. The input tag is an empty element so it has no closing tag. Inside the input tag is usually these 3 attribute-- type, name and value.

Data Processing

Whenever someone fills a HTML form and click the submit button, the form data will be passed to either the URL or to the server. The HTML <form> tag has an attribute named "method" to specify where we want the data to go. There are 2 types of actions, "GET" and "POST" (case sensitive, beware). "GET" will send the data into the URL directly and the data is visible to the user. This method of sending data is not safe at all (with exceptions, of course). Usually, we send the data to the server through "POST" so that no one can see the data except the programmer, the programmer's program and the server itself, sometimes a good hacker too :)

Languages Used for Data Processing

The most common language used is PHP (Hypertext Preprocessor, it used to be Personal Home Page). To run PHP, what we usually do is to download a virtual server (XAMPP if you're using Windows). XAMPP supports several languages specifically for web programming--PHP, SQL (with a database called MariaDB or MySQL) and Perl for CGI programming. Perl used to be king of all the languages during the 2000 period. It's still alive and it's awesome!

PHP
This is the most common language used to interact with the form data. PHP is a programming language.
SQL
"Structured Query Language" is used to interact with the database(DB). After processing data from a form, these data can be stored into the server, we call it the database. SQL can be embeded with PHP through the functions starting with "mysql_*" or "mysqli_*" "mysqli_*" is more secure.
